Modello Client-Server



| Che cos’è?

| L’architettura client/server è un modello di elaborazione in cui il server ospita, consegna e gestisce la maggior parte delle risorse e dei servizi che devono essere consumati dal client.



|Caratteristiche del client e del server

| Il Server può essere definito come il fornitore del servizio e generalmente possiede un indirizzo di IP statico. Se un server è in comunicazione multicast potrebbe riceve troppe richieste contemporaneamente ed entrare quindi in uno stato di congestione,quindi è necessaria la virtualizzazione della risorsa mediante una server farm, ossia assegnare al servizio un unico hostname utilizzando più indirizzi IP, trasparenti rispetto ai client, sui quali vengono dirottate le richieste di connessione.


| Il client al contrario del server può essere visto come lo sfruttatore del servizio e possiede un IP generalmente dinamico. Il client non può comunicare con gli altri client ma solo con il server



| Come funziona?

| 1) Il client manda una richiesta al server

| 2) Il server (perennemente in attesa) riceve la richiesta

| 3) Il server esegue il servizio richiesto (né può eseguire diversi nello stesso momento)

| 4) Il server manda una risposta ed eventualmente dei dati al client.

| 5) Il client riceve la risposta è gli eventuali dati.


| Esempi di servizi tipici:

| HTTP e HTTP utilizzati per la trasmissione d’informazioni sul web.

| FTP utilizzato per il trasferimento di dati tra diversi host.

| Telnet utilizzato per il controllo di computer da remoto.



| Modalità di comunicazione

| Unicast: il server comunica con un solo client per volta e accetta una solo richiesta di connessione per volta.

| Multicast: al server possono essere connessi e possono comunicare più client contemporaneamente.
